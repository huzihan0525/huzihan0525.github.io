What is Machine Learning

Machine Learning: A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E – Mitchell

由此可见，机器学习便是通过经验来完善系统自身的性能。根据不同的任务T，不同的性能评估方法P，不同的训练数据(经验)E，可将机器学习分为很多类。而采用概率的观点的优势则是可以在存在不确定性的情况下完成最优决策。
机器学习大致可分为监督学习，非监督学习，强化学习。

Supervised Learning 监督学习

  • Task T: 学习一个映射(mapping) ，其中输入称作features, covariates, predictors，exogenous variable. 输出称作label, target, response, endogenous variable
  • Experience E: 输入-输出对 , 称作训练集(training set)，N为样本量
  • Performance Measure P: 取决于输出的种类
    根据不同的输出结果，我们可以把监督学习分为 Classification和 Regression.

Classification 分类问题

  • 输出空间，其中C是无序、互斥的类别，此类问题又称作 Pattern Recongition.

Overfitting and Generalization 过拟合和泛化

过拟合指模型过于复杂，假设过于严格，对训练数据具有较为完美的拟合，为验证是否发生过拟合。假设已知分布，计算在真实分布的理论期望损失(theoretically expected loss/ population risk).

理论损失与训练损失之间的差值称为泛化误差(generalization gap)。但实际情况中，真实分布未知，可以用测试集 (test set)计算出的 test risk代替 population risk，我们要求的模型是在测试集上损失最小的。如图，是不同自由度的多项式回归参数模型，可以看出测试损失先减后增，呈现”U”字形曲线，上升的阶段则发生了过拟合，最终会选择 D=2 的多项式回归模型。

除此之外，还需要验证集(validation set)来进行模型选择(model selection)

没有免费午餐定理: 没有最好的模型，所有的模型都是有效的。任何模型对所有真实问题（假设真是问题f均匀）的误差期望相同。因此需要根据具体的问题来选择算法\模型。

Unsupervised Learning 无监督学习

  • Task T: “Make Sense of” data，并没有被很好的定义，用以”解释”高维的输入数据
  • Experience E: 输入，没有对应的输出

从概率的角度，无监督学习是拟合无条件概率模型 ，用以生成新的数据x；监督学习则是拟合一个条件概率模型，用以指明输入对应的输出分布

Evaluating unsupervised learning 评估无监督学习

对于没有ground truth的无监督学习，如何评估模型的性能呢？

  • 看作概率密度估计，计算模型在测试数据集上的NLL，，NLL越小模型则越好，也就是说，好的模型对于新的数据(来自测试集)应该产生分配大的概率密度
  • 将无监督学习到的表征作为下游监督学习的输入或特征，可以减少标注数据的使用量。例如之前提到的鸢尾花的分类，可以采用原图片像素或提取到的四个特征作为输入，显然后者可以提高样本的学习效率。
  • 可解释性，这需要进行实验以及与外界的交互(例如相关领域专家的判断)

Clustering 聚类

在数据中寻找类似的点，并没有一个正确的类别数目，因此需要在模型复杂度和拟合数据间做权衡。

Latent factors 潜在因子

假设高维输入产生于隐藏的低维潜在因子(latent factors) ， ，我们常假设较简单的先验概率模型 。

当使用线性模型时， ，此模型为因子分析 (factor analysis/ FA)，与线性回归类似，不过只有观测数据 ，而不是 。当 时，模型转化成为主成分分析 (Principle component analysis /PCA)

当不限制线性模型时， ，这样的模型参数很难被训练，此时可利用变分自编码器 (Variational Auto-Encoder /VAE)来近似

Self-supervised learning 自监督学习

通过从大规模的无监督数据中挖掘自身的监督信息 ， 则为自动构造的监督信息，此监督信息表征可以用于下游的的监督学习任务中，这样就不需要推断潜在因子z。

Reinforcement Learning 强化学习

强化学习中，系统或智能体(agent)应学习如何与外界环境交互，这通过策略(policy)来完成，需要智能体对输入x做出反应动作。

  • 监督学习是 Learning with a teacher，每一次学习老师都会告诉你下一步最优的方向(梯度下降)；
  • 强化学习是Learning with a critic，只会得到一个偶然的奖励 /惩罚 (reward/ punishment)

然而得到的奖励是偶然的，智能体很难清楚哪一步动作会对此奖励负责。对于少量的奖励信号的信息的情况，可以通过其他信息来源获取信息，例如通过专家示范 (expert demonstrations)。

Data 数据

  • 小型图片数据集：MNIST(手写数字0-9), EMNIST(手写a-z, A-Z, 0-9),Fashion-MNIST(不同种类的服饰),CIFAR(60k张常见类别的图片)
  • 大型图片数据集：ImageNet(14M张20,000类的RBG图像)
  • NLP: 文本分类，机器翻译，seq2seq任务(文件总结，问题回答等)，语言模型(无条件生成模型)

Missing data

输入一部分有标记的数据，一部分未标记的数据，此类问题为半监督学习(semi-supervisor learning)。
在此仅考虑输入特征丢失的问题，利用含01的矩阵M存储输入数据信息，即代表n实例的d特征丢失。代表含特征的输入，代表丢失特征的输入，代表输出标记矩阵

  • ，称为 missing completely at random (MCAR)
  • ，称为 missing at random (MAR)
  • 其余称为 not missing at random (NMAR)

对于MCAR和MAR问题，可以忽略数据丢失机制；对于NMAR问题，需要为数据丢失机制建模。
常常假设为MAR，但一些判别模型 (discriminative model, 如DNN)，也不能直接使用输入数据。可以采用均值填充(mean value imputation)的方法，或者训练另一个生成模型 (generative model)来填充丢失的输入。
